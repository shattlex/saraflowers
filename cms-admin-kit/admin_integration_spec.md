# Мини-ТЗ для подключения универсальной админки

## Задача
Подключить существующую универсальную админ-панель к новому проекту так, чтобы контент сайта редактировался через админку и корректно отображался на публичной части.

---

## Требования

### 1. Подключение админки
- Подключить файлы:
  - admin.html
  - admin.css
  - admin.js
  - content-store.js
- Админка должна работать отдельно от публичного сайта
- admin.js подключать как ES module

---

### 2. Единая структура данных
Использовать общий data model:

- siteName
- pages[]
- media[]

Страница:
- id
- slug
- title
- type
- inNav
- blocks[]

Поддерживаемые блоки:
- hero
- products
- sectionTitle
- textImage
- text
- image

---

### 3. Рендер сайта
- Рендерить страницы из content.pages
- Навигацию строить по pages с inNav = true
- Активную страницу определять по slug
- Каждый блок рендерить через отдельный renderer
- Неизвестные блоки не должны ломать сайт (fallback)

---

### 4. Работа с изображениями
- Поддержка:
  - URL
  - media:<id>
- media:<id> резолвить через content.media
- При отсутствии изображения использовать fallback

---

### 5. Совместимость
- Не переписывать админку
- Сохранить текущую логику CRUD
- Адаптировать только интеграцию

---

### 6. Хранение данных
- Поддержка localStorage
- Уникальный storage key для каждого проекта
- Если есть backend — использовать API
- Если нет — fallback на localStorage

---

### 7. Проверки
После интеграции проверить:

- страницы создаются и открываются
- slug корректно работает
- блоки отображаются корректно
- изображения работают (URL + media)
- навигация корректная
- изменения из админки видны на сайте
- нет падений при пустых данных

---

## Короткая версия для Codex

Подключи готовую админ-панель как модуль. Используй структуру siteName / pages / media. Рендери сайт из этих данных. Поддержи блоки hero, products, sectionTitle, textImage, text, image. Навигацию строй из pages. Активную страницу определяй по slug. Изображения: URL или media:<id> через media[]. Не переписывай админку — только адаптируй интеграцию. Добавь fallback и обработку ошибок.
